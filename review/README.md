# Archive Triage & Review UI

A local-only tool for reviewing, categorizing, and transcribing archived files — text documents, scanned handwritten notes, images, and old formats. Designed for keyboard-driven triage at speed, with state tracking that compounds over time.

**This entire directory is gitignored.** Nothing here reaches the public repo or the deployed site. Raw scans, categories, transcriptions, and the file index all stay local.

## Quick Start

```bash
# 1. Generate the file index (scans the review/ directory tree)
node review/generate-index.js

# 2. Start the server
node review/serve.js

# 3. Open in browser
# http://localhost:3456
```

## Adding Scanned Notes

Drop scanned images into a subdirectory:

```
review/
    scans/              ← your scanned/photographed pages
        notebook-01/
            page-001.jpg
            page-002.jpg
        loose-notes/
            idea-sketch.png
    ...existing archive files...
```

Then re-run `node review/generate-index.js` to pick them up.

**Supported image formats:** .jpg, .jpeg, .png, .webp, .gif, .tiff, .tif, .bmp, .heic, .heif

**Supported text formats:** .txt, .md, .rst, .org

**Documents** (.doc, .docx, .odt, .rtf, .pdf): displayed if a `.converted.md` version exists alongside them, otherwise shown without preview.

## The Review Interface

### Layout

- **Images** get a split pane: scan on the left, editable transcription on the right
- **Text files** show content in a single pane (split pane appears if you start typing notes)
- **Documents without preview** get a split pane with a placeholder on the left and a notes area on the right

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `1` | Categorize as Fragment |
| `2` | Categorize as Idea |
| `3` | Categorize as Deliria |
| `4` | Categorize as Desiderata |
| `5` | Categorize as Publishable |
| `6` | Categorize as Reference |
| `0` | Categorize as Deletable |
| `←` `→` | Navigate previous / next |
| `Space` | Jump to next uncategorized file |
| `T` | Focus the transcription/notes pane |
| `H` | Toggle history log for current file |
| `Escape` | Leave text input and return to keyboard navigation |

### Categories

- **fragment** — incomplete thought, worth keeping
- **idea** — developed concept with potential
- **deliria** — stream of consciousness, dream logs
- **desiderata** — wishlists, goals, wants
- **publishable** — material suitable for the website with editing
- **reference** — useful but not original writing
- **deletable** — no value

### Filters

Filter by category, file type (Image / Text / Document), or search by filename. Filters combine with search.

### Image Viewing

Click the image to cycle zoom levels (100% → 150% → 200% → 50%). Or use the zoom buttons at the bottom of the image pane.

### Transcription

The right pane is an editable text area. Use it for:
- Manual transliteration of handwritten notes
- Pasting OCR output for comparison/correction
- Adding notes about the scan ("page torn", "see also notebook-02 page 15", etc.)

Transcriptions auto-save after 800ms of idle typing. Status indicator shows saved/unsaved state.

## Data Format

All categorizations, transcriptions, and history are stored in `categories.json`:

```json
{
  "scans/notebook-01/page-001.jpg": {
    "category": "idea",
    "transcription": "The key insight is that...",
    "log": [
      { "action": "categorized", "value": "fragment", "at": "2026-03-20T10:00:00Z" },
      { "action": "categorized", "value": "idea", "at": "2026-03-20T10:05:00Z" },
      { "action": "transcription_updated", "at": "2026-03-20T10:10:00Z" }
    ]
  }
}
```

Each file entry tracks:
- **category** — current classification (null if uncategorized)
- **transcription** — text content (null if none entered)
- **log** — timestamped history of all state changes

Data also saves to browser localStorage as a fallback.

### Export

Click "Export JSON" to download a snapshot with summary statistics.

## Connecting to the Publishing Pipeline

Items marked **publishable** are candidates for the essay drafts pipeline:

1. Review and categorize in the triage UI
2. Mark promising items as "publishable"
3. Write/expand in `~/Writing/essays/`
4. Stage to drafts: `npm run stage essay-name`
5. Preview at unlisted URL
6. Publish: `npm run publish-essay essay-name`

See [docs/ESSAYS.md](../docs/ESSAYS.md) for the full publishing workflow.

## File Index

`file-index.json` is generated by `generate-index.js`. Each entry contains:

```json
{
  "path": "scans/page-001.jpg",
  "name": "page-001.jpg",
  "ext": ".jpg",
  "type": "image",
  "size": 524288,
  "modified": "2026-03-20T...",
  "content": null,
  "convertedFrom": null
}
```

The `type` field is one of: `image`, `text`, `document`.

Text file content is embedded in the index (first 15KB). Images are served directly by the local server. Re-run the generator whenever you add or remove files.

## Architecture

```
review/
    index.html          ← Single-page triage UI
    serve.js            ← Local HTTP server (port 3456)
    generate-index.js   ← Directory walker, builds file-index.json
    categories.json     ← Persistent state (auto-generated, do not edit)
    file-index.json     ← File manifest (auto-generated)
    _manifest.md        ← Catalog of original archive contents
    README.md           ← This file
    scans/              ← Your scanned images (create as needed)
```

Everything is local-only. The server is a minimal Node.js HTTP server with no dependencies — just `node serve.js`.
